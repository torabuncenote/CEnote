高 登録の部分失敗で Auth アカウントが孤立し、モーダルが「登録中...」で固まる index.html:doRegister
不具合

createUserWithEmailAndPassword は成功したが fbDB.ref('/users/'+uid).set() が拒否された場合（RTDBルール等）、Auth にはアカウントが作られたのに /users レコードが無い状態になる。.then 連鎖が走らず signOut/delete/closeRegModal も発火しない。

再現

新ユーザーは管理パネルに現れず管理不能。モーダルは「登録中...」のまま、第2アプリインスタンスがセッション内にリークする。

✅ 修正案：/users 書き込み失敗時に作成済み Auth ユーザーを cred.user.delete() でロールバックし、catch でボタン状態を必ず復帰する。

中 signOut() を await せず delete() するため孤立セッションが残りうる index.html:doRegister
不具合

第2アプリは既定で LOCAL 永続化。signOut() は Promise を返すが、解決前に secondApp.delete() を同期的に呼ぶとサインアウトが中断され、新規ユーザーの認証情報が IndexedDB に残る可能性がある。同一 FB_CFG ゆえ永続化キーが主アプリと衝突しうる。

再現

低速回線での登録後、残存セッションが非決定的に復元され、誤った currentUser やログイン不整合（要キャッシュ削除）として表面化することがある。

✅ 修正案：signOut().then(()=>secondApp.delete()) と連結する。可能なら登録専用に setPersistence(NONE) を使う。
